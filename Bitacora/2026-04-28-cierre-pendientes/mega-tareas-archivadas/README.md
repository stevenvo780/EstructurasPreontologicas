# Tareas del equipo

Esta carpeta reúne el **backlog duro** del proyecto: no las mejoras cosméticas, no los retoques de estilo, no la documentación amable, sino aquello que realmente separa una tesis filosófica prometedora de una teoría fuerte, defendible y eventualmente científicamente operable.

## Principio de organización

La carpeta está dividida por peso real del trabajo.

### Tareas gigantes del equipo

Son tareas que exigen meses, lectura intensiva, diseño metodológico, comparación con rivales, formalización más severa o producción de corpus y validación. Estas **no** son tareas decorativas: son las que deciden si el proyecto se vuelve serio o se queda en arquitectura verbal.

### Tareas documentales delegables a IA

Son tareas más pequeñas, acumulativas y documentales. No son irrelevantes, pero no deben comerse la energía estratégica del equipo. Estas pueden ser absorbidas por soporte documental, asistentes o flujos de automatización.

## Diagnóstico de fondo

La tesis ya tiene algo valioso: un núcleo conceptual fuerte. Pero todavía no tiene lo que un proyecto grande necesita para sobrevivir fuera del entusiasmo interno:

- un objeto principal bien recortado;
- una teoría científica desarrollada con criterios de éxito y fracaso;
- una traducción robusta de obras, tradiciones o dominios completos;
- un aparato comparativo cruel con teorías rivales;
- una estrategia de validación suficientemente dura como para que el proyecto pueda perder.

## Regla de oro

> Si una tarea no aumenta poder explicativo, poder discriminativo o poder de contraste, probablemente es secundaria.

## Estructura de prioridades

- `00–06`: tareas gigantes del equipo.
- `90+`: tareas documentales, auxiliares o delegables.

## Veredicto práctico

Si el equipo quiere convertir este proyecto en algo grande, debe asumir que las tareas aquí listadas no son accesorios: son el trabajo real.
